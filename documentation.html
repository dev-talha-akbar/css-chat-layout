HTML Guide

  • Chat Container

    It all starts with a container div with an id of chat. Once, you have created this div. Your HTML will look like this:

    <div id="chat"></div>
    The chat container makes sure that our chat layout sticks to the bottom of the page.

    Add a class of sidebar to chat container if you want the user list tab to become a sidebar on larger screens.

  • Tabs Container

    Now, we have to make an ul (unordered list) inside our chat container with an id of tab-list. Once you have added this ul, your HTML will look like this:

    <div id="chat">
        <ul id="tab-list"></ul>
    </div>
    The ul#tab-list (unordered list) ìs just a container for our tabs.

  • Understanding Tabs

    What is a tab?

    A tab looks like this:

    Each tab can have three sections:

    1. The header
    2. The body
    3. The footer

    Tab

    Each tab is just a li (list item) with a class of tab.

    Container

    The sections in the tab are made flexible by using table based behaviour. A main container (A div with a class of container) which adds table behaviour in the first place is added to the tab.

    Rows

    Then we define a row for each section (A div with a class of row and a section specific class like row-header is added to the main container).

    Sections

    The header, the body and the footer are added to their respective rows.

    Content

    The contents of the header, the body and the footer are added by making a div with a classname of content (This div is necessary for the body. For consistency, add it to the header and the footer as well). Look at the below code (which defines the structure of a tab) to understand the text better:

    <li class="tab">
        <div class="container">
            <div class="row row-header">
                 <div class="header">
                     <div class="content"><!-- All contents of header here --></div>
                 </div>
            </div>
            <div class="row row-body">
                 <div class="body">
                     <div class="content"><!-- All contents of body here like users or messages --></div>
                 </div>
            </div>
            <div class="row row-footer">
                 <div class="footer">
                     <div class="content"><!-- All contents of footer here like a textarea or input --></div>
                 </div>
            </div>
        </div>
    </li>
    BY DEFAULT, THE TABS ARE ACTIVE. YOU NEED TO TOGGLE THE CLASS inactive TO li.tab TO TOGGLE THE TAB.

      Add this to each list-item we created above with different names in body. You'll have your user-list ready

  • Chat Tabs

    Chat Tabs are also li (list items) with just the class of tab and they have same exact structure as mentioned above. They will contain a header, a body with messages and other cool stuff and a footer with a textarea.

    Chat Tabs will created dynamically when user clicks one of the person from user list.

    How do I create a chat tab dynamically?

    Although, you have to write the JS code yourself. But here is a short guide in the steps form:

    1. You'll bind an event handler to each li in user list.

    2. In that handler, you will define statements for creating a li with a class of tab, fetch messages from the database and adding proper HTML to the li you just created and appending it to ul#tabs-container.

    3. Don't forget to add attach event handler to the elements in the tab as well like you have to make the tab collapsible when user click the header, you have to remove the tab when user click the remove icon somewhere in the header, you have to send message when user presses enter in the textarea e.t.c.

    Please read the FYI note below That will surely bring you some relief.

  • Tab Scroller

    Now, so far we have covered user list tab and chat tabs. But there is one interesting feature you may like to implement just like facebook does. Have you ever experienced opening many chat tabs on facebook? You'll see as the width of the tabs taken together becomes greater than the width of the viewport/screen; some of chat tabs are hidden from the screen and a section appears indicating a number That number of chat tabs are hidden from the screen. I called it as Tab Scroller. A dropdown containing the name of persons whose chat tabs were hidden is shown when you click it.

    The styling of Tab Scroller comes with the css file. If you decide to implement this feature and want to write JS functions for it then you will just make a div with an id of scroller and append it to our div#chat. Now, our HTML will look like below: 

    <div id="chat">
        <ul id="tab-list">
            <li id="user-tab" class="tab">[...]</li>
            <li class="tab">[...]</li>
        </ul>
        <div id="scroller"></div>
    </div>

    It doesn't matter where the HTML of tab scroller appears. It is absolute positioned so, even if you put it before your ul#tab-list. It will still work.

    The inner HTML of tab scroller will look like below:

    <div class="container">
        <div class="row row-header">
           <div class="header">
               <div class="content"><!-- Count of hidden users here --></div>
           </div>
        </div>
        <div class="row row-body">
             <ul class="dropdown"><!-- List of hidden users here --></ul>
        </div>
    </div>

CSS Files

I'm using three CSS files in CSS Chat Layout:

  1. normalize.min.css
  2. chat.css
  3. chat-ie.css

  • normalize.min.css

    The first one is a generic reset file. Many browser interpret the default behavior of html elements differently. By using a general reset CSS file, we can work round this. This file also contains some general styling, such as anchor tag colors, font-sizes, etc. Keep in mind, that these values might be overridden somewhere else in the file.

  • chat.css

    This is the main css file which contains all the styles. You can easily edit the CSS file as you like. It is commented to guide you that which properties are important and why we have used them.

    To ensure that your new styles are applied, make sure that they carry enough "weight" and that there isn't a style lower in the CSS file that is being applied after yours.

  • chat-ie.css

    This is a conditional stylesheet. It will be included if your page is loaded in old IE browsers. It basically hides the chat layout from your website instead of breaking all of the page. The old browsers do not support many properties require to make the layout.

    As this file is to be included conditionally, so its HTML looks a bit different. Like this:

    <!--[if lt IE 7]>
        <link rel="stylesheet" type="text/css" href="css/chat-ie.css">
    <![endif]-->

    The modern browsers will ignore it. But IE will read it. It will read the condition. The condition says to Internet Explorer:

    Bro, if your version is less 8 then include this file.

    If the version is less than 8 then Internet Explorer will say:

    Everything's cool, man. I will include it.

    Then IE will apply whatever styles are in the CSS file. Go ahead and open css file and you will see it has something like this in it:

    div#chat {
        display: none;
    }

    So, basically; The chat layout will be hidden in the IE browsers having version less than 8.

LESS File

A LESS file is also included so, you can edit the CSS Chat Layout as you like. Don't worry if you don't know about LESS; you can still edit the CSS file

There are some important variables in the LESS file. Just by editing those variables; you can control the whole layout.

Javascript

CSS Chat Layout provides you with the layout only but two js files named main.js and elastic.min.js is included.

You can test the important dynamic behaviour with ease. The use of those JS files in production is not recommended. You can quickly include those files in your project to test dynamic behaviour.
